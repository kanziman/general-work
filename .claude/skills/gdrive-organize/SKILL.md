# Google Drive Organize Skill

이 스킬은 사용자가 "구글드라이브 정리해줘" 등의 명령을 내릴 때, 구글 드라이브의 `90_Inbox` 폴더에 업로드되어 있는 파일들을 분석하여 적절한 폴더로 자동 분류 및 이동하고 기록을 남기는 스킬입니다.

## 1. 전제 조건 및 제한 사항
- **업로드**: 사용자가 정리할 파일들을 구글 드라이브의 `90_Inbox` 폴더에 직접 사전에 업로드해 둡니다. 이 스킬은 파일 업로드를 직접 수행하지 않으며, 오직 '정리 및 이동' 기능만 담당합니다.
- **수동 호출 제한**: `disable-model-invocation: true` 설정에 따라, 사용자가 명시적으로 정리 요청을 수동 입력할 때만 동작해야 합니다.

## 2. 안전 규칙 (Safety First)
- **[승인 필수]** 정리 대상 파일과 제안 폴더 매핑 목록을 사용자에게 표로 먼저 보여주고, 명시적인 승인을 얻기 전에는 구글 드라이브 상에서 어떠한 파일 이동(MOVE) 연산도 실행해서는 안 됩니다.
- **[불확실성 처리]** 매핑이 모호하거나 애매한 파일은 임의로 이동시키지 말고 `90_Inbox` 폴더에 그대로 둡니다. 미리보기 표에서 '불확실/미지정' 상태로 표시하여 사용자의 수동 개입을 유도하십시오.

## 3. 폴더 분류 기준
파일명 및 파일 확장자, 파일의 간략한 메타데이터를 기반으로 대상 폴더를 결정합니다:
- **`10_보고서`**: 보고서, 분석 문서, 기획안, 슬라이드 등 업무 문서류
- **`40_메일`**: 이메일 백업 파일, 회신 내역, `.eml` 확장자 등 메일 관련 자료
- **`90_Inbox`에 잔류**: 위 두 분류에 속하지 않거나 애매한 모든 유형의 파일 (사용자에게 직접 경로 지정을 유도)

## 4. 동작 프로세스

### [1단계] 90_Inbox 폴더 조회
- 구글 드라이브의 `90_Inbox` 폴더 내 파일 목록을 읽어옵니다.

### [2단계] 미리보기 제공 및 승인 대기
- 파일명과 확장자 분석을 거쳐 분류 결과를 표 형식으로 정리하여 사용자에게 제시합니다.

   #### 구글 드라이브 정리 예정 표
   | 파일명 | 파일 유형/확장자 | 제안하는 대상 폴더 | 상태/비고 |
   | :--- | :--- | :--- | :--- |
   | `2026-06-25 주간보고서.docx` | 문서 (docx) | `/10_보고서` | 자동 분류 |
   | `RE_회의일정안내.eml` | 메일 (eml) | `/40_메일` | 자동 분류 |
   | `unknown_data.bin` | 알 수 없음 | `90_Inbox`에 잔류 | **불확실 (의견 필요)** |

- **[STOP]** 표를 보여준 즉시 진행을 멈추고 사용자의 최종 검토 및 이동 승인 답변을 기다립니다.

### [3단계] 파일 이동 및 사후 확인 (승인 후)
- 사용자가 승인하면, 구글 드라이브 상에서 파일 이동 연산(예: `MOVE_FILE` 또는 부모 폴더 수정 API)을 수행하여 각 파일을 지정된 대상 폴더로 옮깁니다.
- 이동 완료 후, 각 파일의 부모 폴더 ID/경로가 올바르게 업데이트되었는지 재조회하여 검증합니다.

### [4단계] 구글드라이브 MOC 갱신
- 로컬 옵시디언 보관함 내의 [googledrive MOC.md](file:///c:/Users/acrof/DEV/00.업무지식창고/work_store/00_MOC/googledrive%20MOC.md) 파일을 확인합니다. (없다면 새로 생성)
- MOC의 **'정리 기록'** 표에 아래와 같이 정리 이력을 행으로 추가합니다.

   #### googledrive MOC.md 기록 예시
   ```markdown
   | 파일명 | 구글드라이브 경로 | 링크 | 날짜 |
   | :--- | :--- | :--- | :--- |
   | 2026-06-25 주간보고서.docx | `/10_보고서/2026-06-25 주간보고서.docx` | [드라이브 링크](URL) | 2026-07-23 |
   ```

### [5단계] 요약 보고
- 정리가 완료되면 다음 내용을 요약하여 보고합니다:
  - 실제로 이동 완료된 파일 총 개수
  - 대상 폴더별 분포 수치
  - 불확실하여 `90_Inbox`에 그대로 잔류시킨 파일 목록
