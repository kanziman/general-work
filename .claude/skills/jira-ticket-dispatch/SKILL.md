# Jira 티켓 디스패치 (Jira Ticket Dispatch)

이 스킬은 사용자가 문서(보고서, 회의록 등)나 자연어 입력을 바탕으로 Jira 티켓을 생성해 달라고 요청할 때 동작하는 스킬입니다.

## 트리거 키워드
- "지라/Jira 티켓 만들어줘"
- "이 보고서로 티켓 만들어줘"
- "지라 티켓 생성해줘"
- 기타 Jira 티켓 생성 관련 요청

## 동작 규칙

### 1. 입력 분석
- **문서 참조가 있는 경우**: 사용자가 `@` 기호로 문서를 참조하거나 파일 경로를 제공한 경우, 해당 문서를 읽고 (특히 '다음 주 제안/액션아이템', '향후 계획', 'Action Item' 등의 섹션 위주로) 할 일을 추출합니다.
- **문서 참조가 없는 경우**: 사용자가 자연어로 입력한 할 일 내용을 티켓 생성을 위한 원천 데이터로 사용합니다.

### 2. 티켓 정리 및 구성 규칙 (각 티켓 기준)
- **제목**: 한 줄로 요약된 명확한 실행형 제목 (예: "[HPBIZ] 회원가입 프로세스 UI 개선")
- **설명**: 
  - 티켓 생성을 결정하게 한 객관적인 근거(구체적인 수치, 데이터 등)를 2~3줄 내외로 작성합니다.
  - 출처가 된 문서가 있을 경우, 본문 마지막에 출처 문서명(예: `출처: [[멤버십 보고서 MOC]]`)을 포함합니다.
- **프로젝트**: `HPBIZ` 또는 `BSAI` 중 문맥에 맞는 프로젝트 키를 할당합니다. 판단이 모호할 경우 사용자에게 확인합니다.
- **이슈 타입**: `작업(Task)`으로 고정합니다.
- **라벨**: 내용에서 핵심 키워드를 추출하여 라벨로 등록하되, 사용자가 명시한 라벨이 있다면 이를 최우선으로 적용합니다.
- **담당자**: 기본적으로 비워둡니다(Unassigned). 사용자가 명시적으로 담당자를 지정한 경우에만 설정하며, 타인을 함부로 담당자로 지정하지 않습니다.

### 3. 진행 단계 및 프로세스 (반드시 순서대로 수행)
1. **티켓 미리보기 (Preview)**:
   - 생성 예정인 티켓들의 정보를 마크다운 표(Table) 형태로 정리하여 사용자에게 보여줍니다.
   - 포함 항목: 프로젝트, 이슈타입, 제목, 설명, 라벨, 담당자
2. **승인 요청 및 대기**:
   - 미리보기를 출력한 즉시, **"이대로 발송할까요? 고칠 부분 알려주세요"**라고 묻고 사용자의 답변을 기다립니다. (사용자의 최종 승인이 떨어지기 전에는 절대로 실제 Jira API나 도구를 호출하여 티켓을 생성해서는 안 됩니다.)
3. **피드백 반영**:
   - 사용자가 수정을 요청하는 경우(예: "프로젝트를 BSAI로 바꿔줘", "라벨에 '기획' 추가해줘" 등), 의견을 반영하여 다시 미리보기 표를 제공하고 승인 단계(2번)로 돌아갑니다.
4. **티켓 생성 및 결과 수집**:
   - 사용자가 최종 승인을 하면, 각 티켓을 Jira 시스템에 생성하고 생성된 티켓의 키(Key)와 URL 링크를 수집합니다.
5. **원본 문서 되먹임 (Feedback)**:
   - 만약 원본 문서가 존재하고, 거기서 할 일을 추출한 것이라면, 해당 원본 문서의 항목 옆에 생성된 티켓 링크를 `(MKT-142 링크)` 형태로 업데이트(추가)하여 되먹임을 수행합니다.
6. **최종 보고**:
   - 마지막에 성공적으로 생성된 모든 티켓의 키와 링크 목록을 깔끔하게 정리하여 보고합니다.
