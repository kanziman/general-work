# Google Calendar Event Skill

이 스킬은 사용자가 "캘린더에 넣어줘", "이 회의록에서 일정 잡아줘" 등의 요청을 할 때, 지정된 문서나 대화 본문에서 일정을 추출하여 구글 캘린더에 등록하는 역할을 담당합니다.

## 1. 트리거 키워드
- "캘린더에 넣어줘"
- "이 회의록에서 일정 잡아줘"
- "일정 등록해줘"
- 기타 구글 캘린더 일정 등록 관련 자연어 요청

## 2. 일정 분석 및 정리 기준
문서나 사용자 입력에서 일정을 추출하여 다음 형식으로 정보를 구조화합니다:
- **제목**: 일정의 성격을 드러내는 명확한 요약
- **날짜 및 시간**: 시작 일시 및 종료 일시
  * *시간 정보가 없거나 불분명할 경우*: '종일 일정'으로 기본 설계하거나, 사용자에게 추가 질문을 하여 구체적인 시간을 확정합니다.
- **참석자**: 회의나 미팅의 참여 인원 정보
- **장소**: 오프라인 위치 또는 온라인 회의 링크 등
- **메모(출처)**: 일정의 근거가 되는 원본 문서명 및 세부 사항 기재
- **캘린더 계정 선택**: 사용자의 구글 캘린더 계정이 여러 개인 경우, 일정을 어느 계정/캘린더에 등록할지 사전에 확인합니다.

## 3. 진행 단계 및 프로세스 (순서 엄수)

### [1단계] 일정 추출 및 미리보기 제공
- 입력 데이터에서 추출한 일정 후보군을 아래 마크다운 표 형식으로 정리하여 사용자에게 보여줍니다.

   #### 캘린더 등록 예정 일정 표
   | 제목 | 날짜 및 시간 | 참석자 | 장소 | 캘린더 계정 | 비고 |
   | :--- | :--- | :--- | :--- | :--- | :--- |
   | [회의] 6월 등급UP 배너 1차 시안 회의 | 2026-06-24 14:00 - 15:00 | 정해든 부장, 이서연 대리 | 디자인팀 회의실 | 기본 캘린더 | 출처: [[회의록]] |

### [2단계] 승인 대기 및 질문 (STOP)
- 미리보기 표를 출력한 직후 **"이대로 등록할까요? 고칠 부분 알려주세요"** 라고 명시적으로 질문을 던지고 진행을 멈춥니다.
- **[CRITICAL]** 사용자의 최종 승인이 나기 전에는 절대 Calendar MCP 도구를 호출하여 일정을 생성해서는 안 됩니다.

### [3단계] 피드백 반영
- 사용자가 피드백을 주면(예: "시간을 15시로 바꿔줘", "장소를 줌 미팅으로 설정해줘" 등), 의견을 반영하여 수정된 미리보기 표를 다시 제공하고 승인을 다시 요청합니다.
- **[초대장 발송 여부 확인]**: 참석자 초대 발송(Send Updates) 여부를 사용자에게 사전에 확인하여 승인을 받으십시오.

### [4단계] 일정 등록 및 링크 수집
- 사용자가 최종 승인하면 Calendar MCP 또는 적절한 구글 캘린더 도구를 호출하여 일정을 정식 등록합니다.
- 등록 완료 후 생성된 일정의 URL 링크 및 ID를 수집합니다.

### [5단계] 결과 보고 및 되먹임 (Feedback)
- 성공적으로 등록된 일정 목록과 링크를 사용자에게 최종 보고합니다.
- 만약 원본 문서(회의록, 보고서 등)에서 일정을 추출한 것이라면, 해당 원본 문서의 해당 텍스트 옆에 등록된 일정 링크를 `(캘린더 등록 완료: [일정명](링크))` 형태로 추가하여 되먹임을 수행합니다.
