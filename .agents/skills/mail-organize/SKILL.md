# Mail Organize Skill

사용자가 회사 메일 텍스트를 제공하며 "메일 정리해줘" 등의 요청을 할 때, 이메일을 구조화하여 옵시디언 볼트(`work_store/40_메일/`) 내에 저장하고 스레드를 관리하는 스킬입니다.

## 1. 트리거 규칙
- 사용자가 이메일 본문/내용을 붙여넣고 "메일 정리해줘", "이메일 정리" 등 메일 정리와 관련된 요청을 할 때 실행됩니다.

## 2. 보안 및 금지 사항
- **[CRITICAL]** 메일의 원본 텍스트 및 정리된 내용을 절대로 외부망, 외부 API, 또는 외부 서비스로 유출하거나 업로드하지 마십시오. 오직 사용자 대화 및 로컬 `work_store/40_메일/` 디렉토리 내부에서만 안전하게 처리되어야 합니다.

## 3. 동작 프로세스

### [1단계] 정보 추출 및 정규 제목 생성
1. 입력된 이메일 본문에서 다음 항목을 분석 및 추출합니다:
   - **제목(Subject)**, **발신자(From)**, **수신자(To)**, **날짜(Date)**, **본문 내용**
2. 추출된 메일 제목에서 `Re:`, `RE:`, `회신:`, `답장:`, `Fwd:`, `FW:` 등의 접두사(대소문자 구분 없음, 공백 포함)를 제거하여 **'정규 제목'**을 생성합니다.
   * *예: "RE: [공지] 멤버십 개편안 검토 요청" -> "[공지] 멤버십 개편안 검토 요청"*

### [2단계] 스레드 매칭 검사
1. `work_store/40_메일/` 디렉토리 내에 이미 저장된 마크다운 파일들의 Frontmatter 영역을 검사합니다.
2. Frontmatter 내 `thread` 필드 값 중 이번 메일의 **'정규 제목'**과 일치하는 파일이 있는지 찾습니다.

#### 케이스 A: 기존 매칭 노트가 존재하는 경우
1. 해당 기존 마크다운 노트를 로드합니다.
2. **'🧵 메일 타임라인'** 섹션의 가장 아래에 이번 메일 정보(메일 날짜, 발신자, 요지)를 추가로 누적합니다.
3. 기존 노트의 **'한 줄 요약'**, **'📌 주요 내용'**, **'📅 일정'**, **'✅ 실행사항'** 섹션에 이번 메일의 최신 정보를 반영하여 보강하고 업데이트합니다.
   - 기존 내용과 새로운 메일의 정보가 상충할 경우, **최신 메일 정보를 우선**하여 덮어씁니다.
   - 상충으로 인해 갱신된 부분이 있다면 본문 내에 짧게 변경점 메모를 남겨 기록을 보존합니다.

#### 케이스 B: 기존 매칭 노트가 존재하지 않는 경우
1. `work_store/40_메일/` 폴더가 존재하지 않는다면 새로 생성합니다.
2. 새 노트를 생성합니다.
   - **파일명**: `YYYY-MM-DD 정규제목.md` (여기서 `YYYY-MM-DD`는 메일 수신/발송 날짜 기준)
   - **Frontmatter 설정**:
     ```yaml
     ---
     type: mail
     thread: "정규 제목"
     from: "발신자 이름/이메일"
     to: "수신자 이름/이메일"
     date: YYYY-MM-DD
     related: []
     ---
     ```

### [3단계] 정리 형식 및 템플릿 적용
저장할 노트의 본문은 반드시 아래와 같은 Markdown 구조를 준수해야 합니다.

```markdown
# [정규 제목]

한 줄 요약: (이메일 스레드 전체를 아우르는 직관적인 한 줄 요약)

---

## 📌 주요 내용
- (핵심 요점 1)
- (핵심 요점 2)
- (핵심 요점 3 ~ 최대 6개 작성)

## 📅 일정
- YYYY-MM-DD: 일정/마감/미팅 상세 내용 1
- YYYY-MM-DD: 일정/마감/미팅 상세 내용 2
*(일정이 본문에 명시되지 않은 경우 "해당 없음" 표시)*

## ✅ 실행사항
| 담당 | 할 일 | 기한 |
| :--- | :--- | :--- |
| 담당자 이름 | 구체적인 태스크 정의 | YYYY-MM-DD 또는 기한 설명 |
*(실행사항이 없는 경우 빈 표 대신 "해당 없음" 표시)*

## 🧵 메일 타임라인
- **YYYY-MM-DD** (발신자): 메일 요지 요약 (첫 번째 메일)
- **YYYY-MM-DD** (발신자): 메일 요지 요약 (답장/전달 시 누적 추가)

*(이전 정보와의 변경점이 있는 경우 아래에 기재)*
> 💡 **업데이트 메모 (YYYY-MM-DD)**: [기존 실행사항/일정 변경 내용 기술]
```

### [4단계] 연결 (Wiki Link) 설정
- 메일 내용에 언급된 관련 보고서(예: `work_store/10_보고서/` 내부 파일)나 회의록(예: `work_store/20_회의록/` 내부 파일), 혹은 MOC 등의 문서가 존재할 경우 본문 내에 해당 문서명으로 위키링크(`[[문서명]]`)를 달아 연결합니다.
- 또한 Frontmatter의 `related` 필드에도 관련 문서의 위키링크 형식을 추가합니다. (예: `related: ["[[멤버십 보고서 MOC]]"]`)
